# Architecture — Credit Analytics (O&G Business AR)

---

## 1. Design principles

1. **Causal ordering.** Macro paths are generated first; they drive customer credit quality, which drives overdue rates and credit holds. There is no circular dependency.

2. **One scenario at a time.** Each scenario's AR table is generated completely, streamed to parquet, and freed from memory before the next scenario starts. Peak memory is under 200 MB for the standard scale.

3. **Vectorised over customers.** All computation at each time step is performed across all customers simultaneously using NumPy arrays. The Python loop iterates over time (months), not over customers. Each time-step operation is fully vectorised.

4. **Reproducible by construction.** A master seed deterministically derives one child seed per generation layer. The same command always produces identical output.

## 5. Memory management

The standard scale (2 000 customers × 120 months × 4 scenarios) stays well under 500 MB peak:

| Table | Rows per scenario | Approx. size |
|---|---:|---|
| `trade_credit_terms_m` | 240 000 | ~15 MB |
| `macro_scenario_m` | 480 total | < 1 MB |
| `customer_dim` | 2 000 | < 1 MB |

Each scenario is streamed to parquet via `pyarrow.ParquetWriter` immediately after generation and freed before the next scenario starts.

---

## 6. Parquet streaming writer

Large tables are written using `pyarrow.ParquetWriter` in streaming mode:

```python
writers: dict[str, pq.ParquetWriter] = {}

def _append(name: str, df: pd.DataFrame):
    table = pa.Table.from_pandas(df, preserve_index=False)
    if name not in writers:
        writers[name] = pq.ParquetWriter(str(out_dir / f"{name}.parquet"), table.schema)
    writers[name].write_table(table)
```

This enables efficient filter pushdown when reading:

```python
# Reads only the baseline row group — does not scan the whole file
ar = pd.read_parquet("trade_credit_terms_m.parquet",
                     filters=[("scenario_id", "=", "baseline")])
```

## 7. Scenario architecture

Each scenario is an independent path generated by:

1. A shared correlated AR(1) process (same Cholesky factor, same innovations for baseline)
2. A scenario-specific overlay applied on top — additive or multiplicative shocks over specified month windows
3. Scenario-specific child seeds: `seed["trade_credit"] + hash(scenario_id) % 10_000`

Scenarios share the same customer population but differ in macro/commodity environment, which drives different overdue rates, credit hold frequencies, and AR ageing patterns per scenario.

---

## 8. AR generator: vectorised time loop

The trade credit generator uses a state-tracking vectorised loop:

```python
# Pre-allocate N-dimensional state arrays (one entry per customer)
ar_not_due = np.zeros(N)
ar_1_30    = np.zeros(N)
# ... etc.

for t, month in enumerate(months):       # T = 120 months
    # Macro stress multiplier — scalar from this month's macro table
    macro_mult = compute_macro_stress(macro.loc[month])

    # Commodity risk customers get additional stress when oil price falls
    effective_mult = np.where(commodity_risk, macro_mult_commodity, macro_mult)

    # Vectorised bucket aging (all N customers at once)
    new_90_plus = ar_90_plus * 0.85 + ar_61_90 * (1 - pay_prob_61_90)
    new_61_90   = ar_31_60  * (1 - pay_prob_31_60)
    # ... etc.

    slices.append(build_dataframe(state, month))

return pd.concat(slices, ignore_index=True)
```

The inner loop is over time (T = 120), not customers (N = 2 000). Each time step is fully vectorised over N using NumPy broadcasting.

## 9. Key configuration constants

| Constant | What it controls |
|---|---|
| `MASTER_SEED` | Top-level reproducibility seed (default: 20260605) |
| `SEEDS` | One integer seed per generation layer (5 total) |
| `REGIONS` / `REGION_WEIGHTS` | Geographic distribution of customers |
| `RATING_GRADES` / `RATING_TO_PD` | Standard rating scale with long-run average default rates |
| `MACRO_BASELINE` | Starting levels for 13 macro/commodity variables |

## 10. Extension points

| What to extend | Where | How |
|---|---|---|
| Add a new customer type | `generators/counterparties.py` | Add to `_CUSTOMER_SEGMENTS` with all parameters |
| Add a new macro variable | `generators/macro.py` | Add to `_AR_PARAMS`, `_CORR_PAIRS`, and `config.py` `MACRO_BASELINE` |
| Add a new credit tier | `generators/trade_credit.py` | Add to `_CREDIT_TIERS` with all behavioural parameters |
| Add a new scenario | `generators/macro.py` | Add overlay logic in `_apply_overlay()` |
| Add a new AR metric | `generators/trade_credit.py` | Add column to the output DataFrame in the time loop |
| Add a new validation check | `validation.py` | Add a `_check_*` function and call from `validate_all()` |
| Add a new noise type | `noise.py` | Add a `_*` function and call from `inject_noise()` |
